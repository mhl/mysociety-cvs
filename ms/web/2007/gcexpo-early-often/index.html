  • build and operate websites as commercial contracts for Government We think you should release software and websites early, and release it often Launch early a robust version which has a minimal but useful set of features Watch your customers, encourage feedback from them Improve the site Iterate frequently - we often deploy fixes 5 minutes after being told about the problem Users help you improve the service though real world use As examples of this going to give one grant funded, one commercial National website which collects problem reports and sends them to appropriate local council.
